Applications is the base concept in Akuzo. It represents a single unit of deployment and execution for your code. An application can be a microservice, a serverless function, a containerized application, or any other type of executable code.

## Application types
Akuzo supports various types of applications. Akuzo supports various types of applications.

| Type | Intended use | Public availability |
| --- | --- | --- |
| Worker | Background tasks, event processing, scheduled jobs | No |
| Web | Long-running applications, APIs, microservices | Yes |
| SPA | Single-page applications, frontend applications | Yes |
